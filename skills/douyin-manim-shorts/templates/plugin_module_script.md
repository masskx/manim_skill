## 20-30s Fast-Cut Option

Use this option for AFDA-style mechanism videos when the goal is a stronger Douyin rhythm:

| Time | Beat | Script Goal |
|---|---|---|
| 0-3s | Pain-point hook | Make the viewer feel the practical failure. |
| 3-7s | Problem visualization | Show the feature/tensor/domain mismatch. |
| 7-15s | Mechanism animation | Explain with formula, flow, spectrum, heatmap, gate, residual, or fusion. |
| 15-21s | Code insertion | Show 3-5 lines and highlight the real plug-in call. |
| 21-26s | CTA | Offer PDF notes, PyTorch code, architecture figure, or resource pack. |

## Voiceover Style Rules

- Do not sound like a paper report.
- Avoid "本文提出" and "实验表明" unless the user asks for an academic explainer.
- One sentence should usually be under 14 Chinese characters when possible.
- Use short, direct sentences.
- Pain point first, mechanism second, code value third.

Recommended lines:

```text
不是模型太弱，是特征偏了。
它不改主干，直接插在特征后。
低频看结构，高频看细节。
Gate 决定该补多少。
复制三行，就能接入。
PDF笔记、代码、架构图都整理好了。
```
