# AFDA 即插即用模块短视频脚本

## 视频标题

AFDA：给医学分割加一个频域适配器

## 封面文案

频域适配器

## 0-3 秒 Hook

- 屏幕文字：跨域分割崩？
- 画面：同一特征流从“源域”走到“目标域”，边界细节变红抖动。
- 口播：医学分割一换数据域，边界就容易飘？
- 字幕：跨域分割崩？

## 3-7 秒痛点

- 屏幕文字：风格和细节混了
- 画面：特征图被拆成低频大块和高频边缘，未处理时一起混进 decoder。
- 口播：问题是，成像风格和病灶细节，经常被模型混在一起。
- 字幕：风格和细节混了

## 7-13 秒 AFDA 模块机制

- 屏幕文字：先分频，再筛细节
- 画面：输入特征进入 AFDA，分成 low 和 high；low 生成 gate，high 经过轻量卷积，最后相乘。
- 口播：AFDA 的做法很直接：低频负责判断全局， 高频负责保留边缘细节。
- 字幕：先分频再筛细节

## 13-17 秒插入位置

- 屏幕文字：插在特征后
- 画面：Backbone 后面弹出 AFDA 小模块，再接 decoder 或任务头；旁边局部放大论文架构图中的频域对齐区域。
- 口播：把它插在 backbone 的特征后面，再送进 decoder。
- 字幕：插在特征后

## 17-21 秒代码展示

- 屏幕文字：三行就能用
- 画面：

```python
afda = AFDA(channels=64)  # 通道数对齐
feat = backbone(x)        # 原始特征
feat = afda(feat)         # 频域适配
```

- 口播：代码里最小用法，就是通道数对齐，然后包一层特征。
- 字幕：三行就能用

## 21-25 秒结尾引导

- 屏幕文字：适合跨域医学分割
- 画面：AFDA 模块高亮，输出更稳定的边界线；出现“论文 + 代码 + 结构图”资料提示。
- 口播：如果你的任务也卡在跨域医学分割，可以先试这个适配器。
- 字幕：跨域分割可试
